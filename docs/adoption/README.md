# The adoption kit — bringing habitable to your union

This folder is for the people who introduce habitable to others: organizers running a
workshop, coordinators deciding whether a union should adopt it, and the tenants they
hand it to. The reference docs (the [setup guide](../setup-guide.md), the
[mobile guide](../mobile.md), [key management](../key-management.md), and the
[threat model](../threat-model.md)) explain *how the tool works*. This kit is about
*helping real people start using it together*, calmly and safely.

> **Alpha software — please read this first.** habitable works end to end, but it has
> **not been independently audited or proven in court**, and a recorded screen-reader
> pass and signed app-store binaries are still outstanding. **Do not rely on it for a
> real legal matter yet.** Use this kit to *practice*, to *evaluate*, and to *prepare* —
> not to bet a tenant's case or safety on it today. When that changes, the README and the
> threat model will say so plainly. See
> [Honest limits](../../README.md#honest-limits--what-habitable-does-not-do).

## What's here, and who it's for

| File | For | Use it to |
| --- | --- | --- |
| [workshop-facilitator-guide.md](workshop-facilitator-guide.md) | **Organizers / trainers** running an onboarding session | Run a ~90-minute hands-on workshop (or train another trainer): agenda, prep, a consent/safety segment, and guided practice through install, capture, export, verify, and key backup. |
| [quickstart-en.md](quickstart-en.md) | **Tenants** (English) | A printable one-page get-started: install, capture an issue, what the statuses mean, keep your evidence safe, and the one big warning. |
| [quickstart-es.md](quickstart-es.md) | **Tenants** (Spanish) | The same one-pager in natural, plain Spanish. Print and hand out. |
| [board-risk-briefing.md](board-risk-briefing.md) | **Union boards / decision-makers** | A short, honest risk-and-benefit briefing for deciding whether to adopt: what it protects, what it does **not** do, the residual risks, and what adopting responsibly looks like. |

## How they fit together

- **Deciding whether to adopt?** Start with the
  [board risk briefing](board-risk-briefing.md). Bring it to the people who answer for
  members' safety.
- **Adopting, and running your first session?** Use the
  [facilitator guide](workshop-facilitator-guide.md) to run the workshop, and print the
  [quick-start](quickstart-en.md) ([Spanish](quickstart-es.md)) for everyone to take home.
- **Want the full technical "why"?** Every claim here traces back to the
  [setup guide](../setup-guide.md), [mobile guide](../mobile.md),
  [key management](../key-management.md), and [threat model](../threat-model.md).

This kit is documentation, not legal advice. Work with your tenant attorney or legal-aid
group on what to file and when.
