# Capability and claim ledger

**Current as of:** 2026-07-09. CI verifies every local evidence path below.

This is the short, current source for what habitable may honestly claim. Automated tests
show that a code path behaves as asserted under synthetic fixtures; they do **not** prove
legal admissibility, real-world safety, usability, accessibility conformance, or fitness
for a tenant's case. Dated research, audit, and roadmap documents remain useful context,
but this ledger controls when their historical wording differs from current code.

Status vocabulary:

- **Shipped** — implemented on `main` with a directly linked automated test.
- **Partial** — implementation exists, but the named gap limits the claim.
- **Planned** — no usable implementation exists on `main`.
- **Externally unvalidated** — implementation and automated evidence exist, but the
  outcome requires independent human or domain review that has not happened.
