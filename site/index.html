Sharing is a separate action

One record. Two deliberately different copies.

Habitable keeps the complete tenant-held record on the device. Creating a review copy is a separate step with its own contents and disclosures.

Canary · tenant copy

Kept here

  • Sealed originals
  • Original photo metadata
  • Complete local custody details
Pink · review copy

Leaves only when created

  • Whole-unit record in the current alpha
  • Photo metadata removed by default
  • Bundle a recipient can verify independently

Current limits

The project’s claims have a status, too.

Claim Current evidence Status
The app detects changes to sealed files and custody records. Automated fixity, custody, packet, and verifier tests. Implemented and test-gated
The security and cryptography have been independently reviewed. An independent security and cryptography audit. Not completed
The packet workflow fits housing-law practice. Housing-law review of the legal framing and packet workflow. Not completed
The workflow works for tenant organizations. A real tenant-union or legal-aid pilot with documented outcomes. Not completed
The interface works with major screen readers. A recorded human NVDA and VoiceOver pass, plus signed native distribution. Still open

Integrity checks cannot prove what a photo depicts, who created it, or whether a particular court or agency will admit it. Habitable produces a verifiable, court-organized alpha packet—not a promise about a legal outcome.

Run the alpha locally

There is no hosted case app. The local interface requires uv and Python (3.14).

git clone https://github.com/ChelseaKR/habitable
cd habitable
uv sync
uv run habitable demo
uv run habitable app

AGPL-3.0; the standalone verifier also carries Apache-2.0 permission.
